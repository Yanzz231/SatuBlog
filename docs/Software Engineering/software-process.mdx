# Software Process

**Software Process** => kumpulan aktivitas (**activities**), tindakan (**actions**), dan tugas (**tasks**) yang dilakukan untuk menghasilkan sebuah produk, yaitu _software_ beserta dokumen-dokumen pendukungnya (**artefak**).

### 3 Komponen Utama Software Process

1. **Aktivitas (Activity)**
   - Tujuan atau objektif dalam proyek
   - Contoh: Berkomunikasi dengan stakeholder communication, menyusun perencanaan proyek, dan penjaminan kualitas (quality assurance).

2. **Tindakan (Action)**
   - Kumpulan dari beberapa tugas yang menghasilkan sebuah produk kerja utama.
   - Contoh: Membuat desain arsitektur yang menghasilkan sebuah model arsitektur sistem.

3. **Tugas (Task)**
   - Pekerjaan yang lebih kecil dan terdefinisi dengan jelas dan menghasilkan hasil yang nyata (tangible outcome).
   - Contoh: Menulis kode untuk unit test atau meninjau sebuah user story.
